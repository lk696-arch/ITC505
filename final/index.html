Revenge of Lights Out

ITC 505 — JavaScript Final Project

Turn out all the lights!
Moves 0
How to play: Click any square to toggle its light and the lights directly above, below, left, and right of it. The goal of the puzzle is to turn all the lights off (every square dark).

Each time the page loads or you click New Game, a new randomly generated but solvable board is created by simulating random legal moves from an all-off starting state.

Addendum: Design, Process, and Challenges

My implementation of Revenge of Lights Out is built around a dynamic JavaScript-generated 5x5 grid. Each square is a button element with a data-row and data-col attribute so that the game logic can easily translate user clicks into board coordinates. The game state is stored in a two-dimensional JavaScript array of booleans, where true represents a light that is on and false represents a light that is off. When the user clicks a square, the code toggles that square and its orthogonal neighbors using a single makeMove() function, which keeps the logic clear and easy to extend.

To satisfy the requirement that the starting configuration is random but solvable, I start from an all-off board and then simulate a series of random legitimate moves. Because every board state is just a combination of valid moves from the all-off configuration, the final pattern must be solvable by reversing some sequence of moves. A move counter tracks how many clicks the player uses, and after each move the game checks whether all entries in the state array are false; if so, a popup appears with window.alert("You win!") and the game automatically starts a fresh randomized puzzle. This matches the classic Lights Out rule of “turn off all the lights” and gives immediate feedback when the puzzle is solved.

Visually, I leaned into the “revenge” theme with a dark, neon-inspired aesthetic. The overall page uses the Quicksand font for body text and the retro Press Start 2P font for the main title and addendum heading. Both fonts are served through Google Fonts using the standard <link>-based approach in the document <head>, which satisfies the lab’s “font-craziness” requirement. The board uses CSS Grid to keep the layout responsive, and CSS transitions provide smooth changes between the bright “on” state and the dark “off” state. One challenge I ran into was keeping the logic and styling separated: the solution was to have JavaScript only toggle an is-off class on each square and let CSS handle colors, shadows, and hover effects. This made it much easier to tweak the visual design after the core game logic was finished.

Overall, this project reinforced how event listeners, data attributes, and arrays can work together to manage interactive state on a web page. It also showed how a simple rule set (toggling neighbors) can still produce interesting and challenging puzzles when combined with randomness and clean UI feedback.
